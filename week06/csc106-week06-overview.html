Week 6: File Permissions, Ownership, and Networking Fundamentals

CSC 106 - Introduction to Linux II

Welcome to Week 6. This week brings together two topics that are essential for anyone managing a Linux system: controlling who can access files and understanding how your server communicates on a network.

You have spent the past several weeks building your VM, exploring the kernel, managing services, writing scripts, and setting up users and groups. Now you will see how file permissions tie directly into the user and group structure you worked with last week, and you will take your first real look at your server's network configuration. Both of these skills feed directly into Week 7, where you will configure a static IP, set up a web server, and lock things down with a firewall.

This is a lighter week in terms of point value (30 required points), which gives you some breathing room between Mini-Project 1 and the heavier server work coming next week. Use that space to really understand permissions and networking fundamentals. They come up constantly in real-world Linux administration.

Learning Goals

By the end of this week, you should be able to:

  • Change file and directory ownership using chown and chgrp
  • Set and interpret file permissions using both symbolic and numeric (octal) notation with chmod
  • Explain how Linux permission bits (read, write, execute) apply differently to files and directories
  • Use networking commands (ip addr, ip route, ping, dig, ss) to inspect your server's network state
  • Describe what each networking command reveals about your system's connectivity and services
  • Connect file ownership and permissions to the user/group concepts from Week 5

Key Concepts and Themes

Pay attention to these ideas as you work through this week's materials:

Permissions and Ownership

Every file and directory on a Linux system has an owner, a group, and a set of permission bits. Understanding chmod, chown, and chgrp is not optional for system administration. You will use these constantly.

Octal vs. Symbolic Notation

You can express permissions as rwxr-xr-- or as 754. Both say the same thing. Get comfortable reading and writing both forms, because you will encounter each in documentation and scripts.

Network State Inspection

Commands like ip addr, ip route, ping, dig, and ss let you answer basic questions: What IP does my server have? Can it reach the internet? What services are listening? These are your first diagnostic tools.

Connecting Users to Files

Last week you created users and groups. This week you see why that matters: permissions are enforced based on who owns a file and what group it belongs to. The two topics are deeply linked.

Looking Ahead

Next week (Week 7) is the most hands-on server configuration week of the course. You will assign a static IP address using Netplan, install and configure Nginx as a web server, set up a firewall with ufw, configure SSH key authentication, and create a cron job. It is the largest single hands-on assignment in the course (45 points), and it pulls together nearly everything you have learned.

The permissions knowledge from this week matters immediately: web server files need correct ownership and permissions to be served. The networking fundamentals you practice this week (checking your IP, verifying connectivity, seeing what ports are open) become your troubleshooting toolkit when configuring services next week.

If you invest time this week in really understanding how ip addr and ss work, you will save yourself debugging time in Week 7. That is a good trade.
