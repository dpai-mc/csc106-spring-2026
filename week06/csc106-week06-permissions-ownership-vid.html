Linux File Permissions and Ownership

This video introduces Linux file permissions and ownership — the system that controls what each user can read, write, or execute on files and directories. The instructor demonstrates how to read ls -l permission strings, then uses chmod in both symbolic and numeric modes to change permissions. Ownership changes with chown are shown, including how to assign a group as co-owner of a shared project directory. The video closes with a practical shared-directory walkthrough using a devteam group with users alice and bob.

Commands Used in This Video

Commands appear in the order they are demonstrated in the video.

  1. ls -l Long listing — shows permissions, owner, group, size, and date for each file
  2. ls -la Long listing including hidden files (names starting with a dot)
  3. ls -la /tmp Long listing of the /tmp directory to see directory permission strings
  4. touch testfile.txt Create an empty test file to practice permissions on
  5. ls -l testfile.txt View the permissions of a single file
  6. chmod u+x testfile.txt Add execute permission for the owner (symbolic mode)
  7. chmod g-w testfile.txt Remove write permission from the group (symbolic mode)
  8. chmod o-r testfile.txt Remove read permission from other (everyone else)
  9. chmod ug=rw,o=r testfile.txt Set exact permissions for multiple groups at once using = (symbolic mode)
  10. chmod 755 testfile.txt Set permissions numerically: rwxr-xr-x (owner full, group/other read+execute)
  11. sudo groupadd devteam Create a new group named devteam
  12. sudo useradd -m -s /bin/bash alice Create user alice with a home directory and bash shell
  13. sudo useradd -m -s /bin/bash bob Create user bob with a home directory and bash shell
  14. sudo usermod -aG devteam alice Add alice to the devteam group (append, keeping existing groups)
  15. sudo usermod -aG devteam bob Add bob to the devteam group
  16. sudo mkdir /opt/project Create a shared project directory
  17. sudo chown root:devteam /opt/project Set root as owner and devteam as group owner of the directory
  18. sudo chmod 775 /opt/project Give owner and group full access; others read+execute only
  19. sudo -u alice touch /opt/project/alice-file.txt Run a command as alice — creates a file in the shared directory
  20. ls -l /opt/project Verify ownership and permissions of files in the shared directory
  21. sudo chown alice:devteam /opt/project/alice-file.txt Change the file's group ownership to devteam so all group members can access it
  22. sudo userdel -r alice Delete user alice and remove her home directory (-r is recursive)
  23. sudo userdel -r bob Delete user bob and remove his home directory
  24. sudo groupdel devteam Delete the devteam group
  25. sudo rm -rf /opt/project Remove the shared directory and all its contents
