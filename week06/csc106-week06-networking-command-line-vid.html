Inspecting Your Network from the Command Line

This video walks through five essential questions you can answer using Linux networking commands on your Ubuntu Server VM: What is my IP address? What is my default gateway? Can I reach the internet? Is DNS working? What services are currently listening for connections? The instructor demonstrates each command live in the terminal and explains how to interpret the output. This is the foundation you need for the Network Basics Check assignment and for Week 7's server configuration work.

Commands Used in This Video

Commands appear in the order they are demonstrated in the video.

  1. ip addr List all network interfaces and their IP addresses
  2. ip a Shorthand for ip addr — same output, fewer keystrokes
  3. ip addr show enp0s3 Show details for a single named interface (your interface name may differ, e.g., eth0)
  4. ip route Show the routing table — the default via line shows your default gateway address
  5. ping -c 4 8.8.8.8 Send 4 ping packets to Google's DNS server — confirms basic internet connectivity
  6. ping -c 4 1.1.1.1 Alternate connectivity test using Cloudflare's DNS server
  7. ping -c 4 google.com Tests DNS resolution and connectivity together — if this works but ping 8.8.8.8 also works, DNS is functioning
  8. dig google.com Detailed DNS query — shows the question sent, the answer (IP address returned), and the DNS server that responded
  9. sudo apt install dnsutils Install the dig tool if it is not already present on your system
  10. ss -tlnp List TCP listening sockets with numeric port numbers and the process name for each socket
  11. ss -tlp Same as above but shows service names (e.g., http, domain) instead of port numbers
  12. systemctl status nginx Check whether the Nginx web server is installed and running — substitute any service name to check its status
