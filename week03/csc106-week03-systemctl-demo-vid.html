Commands Demonstrated in This Video

Commands are listed in the order they appear in the video and grouped by function. Use this as a reference when following along or completing the systemd Exploration assignment.

Checking service status

  1. systemctl status nginx Shows current state (active/inactive/failed), PID, how long it has been running, and the most recent log lines. The green dot means running; white means stopped; red means failed.
  2. systemctl is-enabled nginx Returns enabled or disabled — whether the service is configured to start at boot. Useful in scripts.
  3. systemctl is-active nginx Returns active or inactive — whether the service is running right now.
Active vs. enabled: These are two separate things. A service can be active (running right now) but disabled (will not start at the next boot). It can also be enabled (will start at boot) but inactive (manually stopped since the last boot). Keep this distinction in mind when reading systemctl status output.

Start, stop, restart, and reload

  1. sudo systemctl start nginx Start a stopped service. No output means it worked — silence is success in Unix.
  2. sudo systemctl stop nginx Stop a running service. Does not affect whether it starts at the next boot.
  3. sudo systemctl restart nginx Kill the process and start a fresh one. Use after configuration changes. There is a brief moment of downtime.
  4. sudo systemctl reload nginx Ask the running process to re-read its configuration without stopping. No downtime, but only works if the service explicitly supports it.
  5. sudo systemctl reload-or-restart nginx Tries reload first; falls back to restart automatically if the service does not support reload. The safer choice when you are not sure.

Enable and disable (boot behavior)

  1. sudo systemctl enable nginx Configure the service to start at boot. systemd does this by creating a symlink inside multi-user.target.wants/. Does not start the service right now.
  2. sudo systemctl disable nginx Remove the service from boot startup by deleting the symlink. Does not stop the service if it is currently running.
  3. sudo systemctl enable --now nginx Enable and start the service in a single command. The --now flag is a handy shortcut you will see frequently in setup guides.

Reading a unit file

  1. systemctl cat nginx.service Display the unit file exactly as systemd reads it. The three sections are [Unit] (description and ordering), [Service] (how to run it), and [Install] (the WantedBy= line that connects it to a target). This is why enable works the way it does.

Shutdown and reboot

  1. sudo systemctl poweroff Shut down and power off cleanly — services stop in dependency order, filesystems unmount safely.
  2. sudo systemctl reboot Reboot cleanly. Equivalent to sudo reboot. Use this instead of forcing the VM off from the VirtualBox or UTM interface.
  3. sudo systemctl halt Halt the CPU without powering off. Less common in practice.
  4. sudo shutdown -h now Halt immediately using the traditional shutdown command (which wraps systemd on Ubuntu).
  5. sudo shutdown -r +5 "Rebooting for updates" Schedule a reboot in 5 minutes and broadcast a message to logged-in users.
  6. sudo shutdown -c Cancel a scheduled shutdown before it runs.

Viewing logs with journalctl

  1. journalctl -u nginx Show all log entries for the nginx unit across all boot sessions, oldest to newest. Press q to quit.
  2. journalctl -u nginx -b Show logs for this boot session only (-b = boot). Usually the most useful option when diagnosing a startup problem.
  3. journalctl -u nginx --since "30 minutes ago" Filter log output by time. Accepts natural language expressions and explicit timestamps.
  4. journalctl -u nginx -f Follow new log entries in real time, like tail -f. Press Ctrl+C to stop. Useful for watching a service as you make changes.
