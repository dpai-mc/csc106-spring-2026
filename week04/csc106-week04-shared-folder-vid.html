Setting Up a Shared Folder in VirtualBox

This video walks through setting up a shared folder between your Windows (or Mac) host machine and your Ubuntu Server VM in VirtualBox. You will install the VirtualBox Guest Additions, add your user to the vboxsf group, and configure a shared folder in VirtualBox settings so files can be transferred between the two systems. This is especially useful for copying scripts into your server without retyping them — once the share is working, you can write a script on your host machine and drop it directly into your VM.

Steps & Commands Used

Prerequisites: Your VM must be running the graphical desktop (from the GUI install video) before starting this process. Steps marked VirtualBox are performed in the VirtualBox application on your host machine, not inside the VM.

Part 1 — Install VirtualBox Guest Additions

  1. VirtualBox With the VM running, click the Devices menu in the VirtualBox window and choose Insert Guest Additions CD Image… The CD image mounts automatically inside the VM. A file manager window may pop up. If it does, right-click the CD icon and choose Properties to see the full mount path (something like /media/your-username/VBox_GAs_7.x.x).
  2. Open a terminal inside the VM, then navigate to the Guest Additions CD:
    cd /media/your-username/VBox_GAs_7.x.x Replace your-username and the version number with what you see in Properties. Capitalization matters — use Tab to auto-complete the path.
  3. Run the Guest Additions installer:
    sudo sh ./VBoxLinuxAdditions.run Enter your password when prompted. The installer sets up shared folder support, clipboard sharing, and dynamic screen resizing. This takes about 30–60 seconds.
  4. Shut down the VM cleanly:
    sudo poweroff You need to power off (not just reboot) before changing the shared folder settings in VirtualBox.

Part 2 — Configure the Shared Folder in VirtualBox

  1. VirtualBox With the VM powered off, open Settings → Shared Folders. Remove any existing shared folder entry, then click the Add icon (folder with a plus sign). Choose the folder on your host machine you want to share (e.g., a folder on your Windows desktop or Documents). Leave Mount Point blank and check Auto-mount. Click OK twice to save.
  2. VirtualBox Start the VM and log in to the graphical desktop. The shared folder should appear under /media/ as a folder named sf_YourFolderName. If you get a “Permission denied” error when you try to open it, continue to Part 3.

Part 3 — Grant Your User Access to the Shared Folder

  1. Open a terminal and add your user to the vboxsf group:
    sudo usermod -aG vboxsf your-username Replace your-username with your actual Linux username. The vboxsf group controls access to VirtualBox shared folders — without this step you will get “Permission denied” errors.
  2. Reboot so the new group membership takes effect:
    sudo reboot You must reboot (not just log out) for the group change to apply to your session.
  3. After rebooting, open the file manager and navigate to /media/sf_YourFolderName to verify access. You should now be able to open, copy, and paste files in both directions. Try creating a test file on your host, copying it to the shared folder, and opening it inside the VM — and vice versa.
Workflow tip: Once the shared folder is working, you can write your Mini-Project 1 script in a text editor on your Windows or Mac host (where copy/paste is easy), save it to the shared folder, then access it inside the VM at /media/sf_YourFolderName/script.sh. Copy it to your home directory with cp /media/sf_YourFolderName/script.sh ~/, then run it from there with chmod +x ~/script.sh && sudo ~/script.sh.
