Free Resources: Bash Scripting Fundamentals

These free web resources cover the advanced bash features you need for Mini-Project 1. You do not need to read all of them cover to cover—treat them as references to consult as you plan and build your script. The Start here items in each section are the highest-value starting points. The Reference items are for looking up specific syntax questions as they come up.

Bash Scripting Fundamentals and Advanced Features

The Advanced Bash-Scripting Guide is the best starting point for the techniques you need this week—it is example-heavy and organized by topic. The Advanced Bash-Scripting Guide is a "manual for self-study, and as a reference and source of knowledge on shell scripting techniques. The exercises and heavily-commented examples invite active reader participation, under the premise that the only way to really learn scripting is to write scripts."

  • Advanced Bash-Scripting Guide ↗
    Start here

    A comprehensive, example-heavy guide to bash scripting. Unlike the official reference manual, this guide explains concepts through worked examples, making it much easier to learn from. The four sections below are the most directly useful for Mini-Project 1—jump to whichever topic you are working on:

    • Functions ↗ — defining, calling, arguments, return values, and local variables
    • Arrays ↗ — declaring, accessing elements, looping, and practical use cases
    • I/O Redirection ↗ — redirecting stdout/stderr to log files; the tee pattern
    • Debugging ↗ — bash -x, set -e, and tracing techniques
  • GNU Bash Reference Manual ↗ gnu.org
    Reference

    The authoritative reference for bash syntax and behavior. Dense and precise—this is not a tutorial. Use it to look up exactly how a specific construct works when you need a definitive answer: special parameters ($#, $@, $?), conditional constructs, arrays, and function syntax are all documented here with full detail. Good to have open in a tab while you are writing.

Error Handling and Best Practices

Mini-Project 1 requires at least one error-handling check. This short article explains a simple pattern that makes scripts fail loudly instead of silently—a habit that saves a lot of debugging time.

  • Unofficial Bash Strict Mode ↗ redsymbol.net — Aaron Maxwell
    Start here

    A short, practical article explaining set -euo pipefail and why adding it near the top of your script helps catch bugs early. With this setting, a command that fails will stop the script immediately rather than letting subsequent commands run on bad state. Recommended reading if you want your script to fail loudly and clearly rather than silently producing wrong results.

Writing a Good README

A README is a required deliverable for Mini-Project 1. This guide explains what makes a README useful and walks you through writing one. The principles apply directly to documenting your post-install script.

  • Make a README ↗ makeareadme.com
    Step-by-step

    A simple, readable guide to writing effective README files. Not Linux-specific, but the advice applies directly to the Mini-Project 1 README deliverable. It covers what to include (description, prerequisites, usage instructions, examples), why each section matters, and how to format the file clearly. The person running your script is often not the person who wrote it—a good README makes that handoff work.
