Key Commands & Concepts Referenced

Note: This video is an orientation overview with no live terminal session. The commands below are the bash scripting techniques discussed in the video, listed in the order they are introduced. They map directly to the Mini-Project 1 requirements.
  1. nano script.sh Create or open a script file in the nano text editor. The .sh extension indicates this is a shell script.
  2. chmod +x script.sh Make the script executable so it can be run directly. Do this once after creating the file.
  3. sudo ./script.sh <argument> Run the script with administrative privileges. Mini-Project 1 must accept at least one command-line argument (e.g., a username).
  4. sudo apt update Refresh the package index. This should be one of the first actions in your post-install script.
  5. sudo apt upgrade -y Upgrade all installed packages. The -y flag automatically answers “yes” to the confirmation prompt.
  6. sudo apt install -y <package1> <package2> Install one or more packages. Your script must install at least three packages.
  7. sudo useradd -m -s /bin/bash <username> Create a new non-root user with a home directory. Mini-Project 1 requires creating at least one user.
  8. sudo usermod -aG sudo <username> Add the user to the sudo group, granting administrative access.
  9. sudo systemctl enable <service> Enable a service to start automatically at boot.
  10. sudo systemctl start <service> Start a service immediately. Your script must enable and start at least one service.
  11. bash -x script.sh Run a script in trace/debug mode — each command is printed before it executes. Useful for finding exactly where a script breaks.
Mini-Project 1 checklist: Your script must include at least one command-line argument, a package index update and upgrade, installation of 3 or more packages, a non-root user with sudo access, enabling and starting a service, a timestamped action log, at least one function, one array, one conditional, comments throughout, and at least one error-handling check. A README file explaining how to run the script is also required.
