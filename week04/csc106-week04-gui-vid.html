Installing a Graphical Interface on Ubuntu Server

This video demonstrates how to install the Xubuntu graphical desktop environment on your Ubuntu Server VM in VirtualBox. The instructor walks through updating the system, installing the XFCE-based desktop with apt, configuring the LightDM display manager, and setting the default boot target to graphical mode. Having a GUI makes it easier to write and test scripts, copy and paste code between your host machine and the VM, and access a web browser from within the server environment. Once installed, you can use the GUI’s built-in terminal emulator to continue working on Mini-Project 1.

Commands Used

Heads up: In the video the instructor first tries sudo tasksel install xubuntu-desktop, which does not work as expected. Skip steps using tasksel and go straight to sudo apt install xubuntu-desktop (step 3 below) — that is the method that works.
  1. sudo apt update Refresh the package index before installing anything new.
  2. sudo apt upgrade -y Upgrade all currently installed packages to their latest versions.
  3. sudo apt install xubuntu-desktop Install the full Xubuntu desktop environment (XFCE-based). This downloads several hundred megabytes and takes a few minutes. When prompted whether to continue, type y and press Enter (or add -y to skip the prompt).
  4. sudo dpkg-reconfigure lightdm Configure the LightDM display manager, which handles the graphical login screen. Run this if the installer did not prompt you to choose a display manager during installation.
  5. sudo systemctl status lightdm Verify that the LightDM service is loaded and enabled. You should see Active: active or enabled in the output.
  6. sudo systemctl set-default graphical.target Set the system to boot into graphical (GUI) mode by default instead of the text console. A symbolic link confirmation message means it worked.
  7. sudo reboot Reboot the VM. If everything is configured correctly, the system will boot into the Xubuntu graphical login screen.
After the GUI loads: Use the Terminal Emulator application inside the GUI to run all your normal Linux commands. The terminal behaves identically to the text-only console you have been using — the GUI just makes it easier to open multiple windows, copy and paste, and use a text editor with a mouse.
