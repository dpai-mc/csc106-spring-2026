CSC 106 · Week 5 · sudo & Security

sudo, Security, and Why Not Root

This video explains the sudo mechanism — how it temporarily elevates your privileges for a single command and why that is safer than logging in as root. Three core reasons are covered: accountability (sudo logs exactly who ran what), damage limitation (a typo as a regular user is far less catastrophic), and the principle of least privilege (users and services should only have the access they actually need). The video also demonstrates adding a user to the sudo group, safely editing /etc/sudoers with visudo, and running quick audit commands to find accounts with empty passwords or unexpected root-level access.

Commands Used in This Video

Commands appear in the order they are demonstrated. Use this list to follow along or review after watching.

Never edit /etc/sudoers directly with a text editor like nano or vim. Always use sudo visudo — it validates the file's syntax before saving and prevents you from locking yourself out of sudo access.
  1. whoami Display your current username. Compare the output with and without sudo to see the difference.
  2. sudo whoami Returns "root" — demonstrates that sudo temporarily makes you root for that one command.
  3. id testuser Check a user's current UID, GID, and group memberships before making changes.
  4. sudo usermod -aG sudo testuser Add testuser to the sudo group. The -aG appends the group without removing existing group memberships. The user must log out and back in for the change to take effect.
  5. id testuser Re-run after the usermod to verify that group 27 (sudo) now appears in the output.
  6. sudo cat /etc/sudoers View the sudoers configuration file. This file controls who can use sudo and what commands they are permitted to run.
  7. sudo grep %sudo /etc/sudoers Find the line granting the sudo group full access. The % prefix means "group." The ALL=(ALL:ALL) ALL rule means members can run any command as any user on any host.
  8. sudo visudo The safe way to edit /etc/sudoers. Opens in the default editor (nano) and validates syntax before saving. Use Ctrl+X to exit without making changes.
  9. sudo cat /var/log/auth.log | grep sudo | tail -5 View the last 5 sudo audit log entries — demonstrates accountability. Each entry records which user ran a sudo command and when.
  10. sudo awk -F: '($2 == "") {print $1}' /etc/shadow Security audit: find accounts with empty (blank) passwords. Any output means a user can log in without a password — investigate immediately.
  11. sudo awk -F: '($3 == 0) {print $1}' /etc/passwd Security audit: find accounts with UID 0 (root-level access). Only "root" should appear. Any other account listed here is a serious security concern.
  12. lastlog Show the last login time for every account on the system. Accounts that have never logged in may be unused — unused accounts increase your attack surface and should be removed.
