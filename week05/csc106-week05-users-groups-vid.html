Users, Groups, and /etc/passwd Explained

This video opens the Linux user database files — /etc/passwd, /etc/shadow, and /etc/group — and walks through every field so you understand what they contain and why they exist. You'll see how to create a new user account with useradd, set a password with passwd, and verify the results in the database files. The video also demonstrates creating a group with groupadd and safely adding a user to it with usermod -aG. These fundamentals are the foundation for file permissions next week and for Mini-Project 2.

Commands Used in This Video

Commands appear in the order they are demonstrated. Use this list to follow along or review after watching.

Important: When adding a user to a group, always use usermod -aG (with the lowercase -a flag). Without -a, the -G flag replaces all existing supplementary groups instead of appending to them — which can accidentally remove sudo access.
  1. cat /etc/passwd Display the full user database — one line per account, seven colon-separated fields each.
  2. grep paid /etc/passwd Look up a specific user entry by username.
  3. sudo cat /etc/shadow View the shadow password file (requires elevated privileges). Contains password hashes and aging fields.
  4. sudo grep paid /etc/shadow Examine your own shadow entry to see the hashed password and aging values.
  5. whoami Display your current username.
  6. id Show your UID, primary GID, and all supplementary group memberships with numeric IDs and names.
  7. groups List only the names of the groups you belong to.
  8. sudo useradd -m -s /bin/bash -c "Test User" testuser Create a new user: -m creates a home directory, -s /bin/bash sets the login shell, -c sets the comment/full-name field.
  9. grep testuser /etc/passwd Confirm the new account was written to the user database.
  10. sudo grep testuser /etc/shadow Check whether the account has a password set (an ! means no usable password yet).
  11. sudo passwd testuser Set a password for the new user (password input is hidden — no characters appear as you type).
  12. ls -la /home/testuser List all files in the new user's home directory, including hidden dot files copied from /etc/skel.
  13. ls /etc/skel View the skeleton directory — whatever is here is automatically copied into every new user's home directory.
  14. sudo groupadd developers Create a new group named developers.
  15. sudo usermod -aG developers testuser Append testuser to the developers group without removing existing group memberships. Always use -aG, never just -G.
  16. id testuser Verify the updated group membership for testuser.
  17. sudo adduser demouser Create a user interactively — prompts for password and name. Easier for one-off use; useradd is preferred in scripts.
