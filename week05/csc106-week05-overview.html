Week 5: Users, Groups, and System Security

CSC 106 - Introduction to Linux II

Welcome to Week 5. You have reached the halfway point of the course, and your Linux system is in solid shape. Over the past four weeks you installed Ubuntu Server, explored the kernel and boot process, managed services with systemd, and wrote a real automation script in Mini-Project 1. That is a lot of ground covered.

This week the focus shifts from building the system to managing who can use it. You will dig into how Linux tracks users and groups, where authentication data is stored, how to configure sudo access, and why user management is the starting point for system security. These are not abstract topics. Mini-Project 2 in Week 8 will require you to set up multiple users with appropriate group memberships and permissions, so the work you do this week is direct preparation for the capstone.

The workload is a bit lighter than Week 4's scripting project. You have one Cisco lab, one discussion, and an optional extra credit exercise. Use the breathing room to read carefully and make sure you really understand what is happening inside /etc/passwd and /etc/shadow. That understanding pays off in every remaining week of the course.

Learning Goals for This Week

By the end of this week, you should be able to:

  • Explain how Linux identifies and tracks users and groups using /etc/passwd, /etc/shadow, and /etc/group.
  • Create, modify, and delete user accounts and groups using command-line tools.
  • Configure sudo access for users and explain why root login is discouraged.
  • Describe how Linux password storage and aging policies work.
  • Articulate why user and group management is the foundation of Linux system security.

This week's work maps to CSLO 1: Install and configure a Linux operating system and CSLO 2: Configure a Linux operating system as a platform for web-based and other applications.

Key Concepts to Watch For

As you work through the reading, videos, and lab this week, pay special attention to these ideas. They come up again in Weeks 6, 7, and 8.

/etc/passwd and /etc/shadow Know what each field means. When you created users in Mini-Project 1, these files are where those accounts actually live.
UIDs and GIDs Linux uses numeric IDs internally. Usernames are for humans. System accounts have UIDs below 1000 on Ubuntu.
useradd vs. adduser Both create users, but they behave differently. Scripts typically use useradd. Interactive setup uses adduser.
The -aG flag When adding a user to a supplementary group, always use usermod -aG. Without the -a, you replace all existing groups. This is a very common mistake.
sudo and least privilege Users should have only the access they need. sudo provides temporary elevation without full root login.
Password hashing Linux never stores plaintext passwords. The hash in /etc/shadow is one-way. Even root cannot read a user's password, only reset it.

Looking Ahead

Next week (Week 6) adds file permissions and ownership on top of what you learn this week. Permissions in Linux are assigned to users and groups, so the concepts from this week are a prerequisite for understanding how chmod, chown, and chgrp work. Week 6 also introduces networking fundamentals, which begins the lead-in to Week 7's server configuration work.

If you set up a developers group this week (or something similar during the Cisco lab), you will see that group again in Week 6 when you create shared directories with group-based permissions. And in Mini-Project 2 (Week 8), you will need at least two non-root users in appropriate groups with sudo access and a shared directory with correct permissions. Everything you practice this week feeds directly into that capstone requirement.

Take the time to understand the "why" behind these commands, not just the "how." That understanding is what separates someone who can follow a tutorial from someone who can troubleshoot a real system.
