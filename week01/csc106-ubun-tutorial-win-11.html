Setting Up Your Linux Environment

This tutorial prepares your Windows 11 computer for Lab 1. You will complete two tasks: install VirtualBox (a hypervisor that lets you run Linux inside a window on your PC), then use it to install Ubuntu Server 24.04 LTS as a virtual machine.

The tutorial is organized as follows: a single video walkthrough covers the entire process from start to finish, followed by two written task sections you can use alongside the video or on their own.

On an older Intel-based Mac? Intel Macs use the same x86_64 processor architecture as Windows PCs. You can follow this guide and download VirtualBox for macOS (Intel) instead of the Windows version — all other steps are the same.

Install VirtualBox

What is a hypervisor?

A hypervisor is software that lets your computer run one or more separate operating systems inside isolated windows, as if each were running on its own hardware. The operating system running inside is called a virtual machine (VM). For this course, your Windows 11 desktop stays exactly as it is — VirtualBox simply adds a layer that runs Linux alongside it. Nothing about your existing system changes.

Oracle VirtualBox is free, open-source, and the standard hypervisor for Windows in this course. You will also install the VirtualBox Extension Pack, which adds support for USB devices, shared clipboard, and better keyboard/mouse integration inside your VM.

Download VirtualBox

Official download page: virtualbox.org/wiki/Downloads
✓ Search for "Oracle VirtualBox download" and go to virtualbox.org. Download both: the Windows hosts installer and the VirtualBox Extension Pack (listed just below the platform installers).
✓ Installer filename starts with VirtualBox- and ends in -Win.exe. The Extension Pack file ends in .vbox-extpack.

Written step-by-step instructions

  1. Open your browser, search for Oracle VirtualBox download, and go to virtualbox.org/wiki/Downloads.
  2. Under VirtualBox platform packages, click Windows hosts to start the download. Save it to your Downloads folder.
  3. On the same page, scroll down to the VirtualBox Extension Pack section. Click the extension pack link (labeled with the version number, e.g., 7.x.x) and accept the license to download it. Save it to the same Downloads folder.
  4. Open your Downloads folder. Double-click the VirtualBox installer (the .exe file) to launch it.
  5. Click through the installer accepting all default settings and locations. If Windows asks about disk space or shortcuts, leave the defaults and continue.
  6. A warning about your network connection being briefly interrupted may appear — this is normal. Click Yes to proceed.
  7. Click Install. If a User Account Control prompt appears, click Yes. Wait about 1–2 minutes for the installation to complete, then click Finish.
  8. Return to your Downloads folder and double-click the Extension Pack file (the .vbox-extpack file). VirtualBox opens automatically and prompts you to install the pack.
  9. In the Extension Pack dialog, scroll down to the bottom of the license text, then click I Agree. Wait about 30 seconds for the install to complete.
  10. Verify the Extension Pack is installed: In VirtualBox, go to Tools in the left sidebar and click Extensions. The Oracle VM VirtualBox Extension Pack should appear in the list with a checkmark showing it is enabled. Check the version number matches the one you downloaded.

Install Ubuntu Server 24.04 on VirtualBox

What is Ubuntu Server — and why no graphical desktop?

Ubuntu Server is a version of Linux built for computers that run services and host files rather than run desktop apps. It has no graphical interface — you interact entirely at the command line. This is exactly how professional Linux servers work in the real world. If the text-only login feels unfamiliar at first, that is completely normal and something you will quickly grow comfortable with over the course of the semester.

Why the ISO version matters: Because you are on a Windows PC (or an Intel Mac), your processor uses the x86_64 (amd64) architecture. You must download the amd64 ISO — the ARM version will not work on your hardware.

Download Ubuntu Server 24.04 LTS

Official download page: ubuntu.com/download/server
✓ Go to the Ubuntu Server download page and click Option 2: Manual server installation, then download Ubuntu Server 24.04.x LTS. This is the correct x86_64 (amd64) version for Windows / VirtualBox.
✓ Filename looks like ubuntu-24.04.X-live-server-amd64.iso — approximately 2.7–3.3 GB. The download may take several minutes; Ubuntu's servers can be slow.
✓ Optional — Verify your download: On the download page, click verify your download to get the SHA-256 checksum. On Windows, use a free tool such as HashCalc (or certutil -hashfile <filename> SHA256 in PowerShell) to generate a hash of the downloaded ISO and compare the first and last few characters to the expected value.

Written step-by-step instructions

Part A — Create the Virtual Machine

  1. Open VirtualBox. Click the New button (blue star icon) to open the new VM wizard.
  2. Enter a name for your VM, for example CSC106-Ubuntu.
  3. Leave the default storage location as is (or change it if you prefer a different drive).
  4. Under ISO Image, click the dropdown and choose Other…. Browse to your Downloads folder and select the Ubuntu Server ISO file (ubuntu-24.04.X-live-server-amd64.iso). Click Open.
  5. VirtualBox will auto-detect the type as Linux / Ubuntu (64-bit). Check the box labeled Skip Unattended Installation so you can walk through every screen yourself.
  6. Click Next. On the Hardware screen:
    • Base Memory: 4096 MB (4 GB) is recommended for a smooth install. Use at least 2048 MB if RAM is limited.
    • Processors: 2 CPUs recommended; 4 if you want a faster install experience.
  7. Click Next. On the Hard Disk screen, set the virtual disk size to 25 GB (the default). This is sufficient for this course.
  8. Click Next, review the summary, then click Finish.
  9. Fix the boot order before starting: With the new VM selected, click Settings → System. Under the Boot Order list, uncheck Floppy and move Hard Disk to the top of the list (above Optical). Click OK. This ensures the VM boots from the hard disk after Ubuntu is installed, and falls back to the ISO during the first install.
  10. Click Start to power on the VM. The Ubuntu installer will boot from the ISO.

Part B — Walk Through the Ubuntu Installer

Use the arrow keys and Enter to navigate the installer. Steps marked 📷 Screenshot are required for the Lab 1 submission.

  1. At the boot menu, select Try or Install Ubuntu Server and press Enter.
  2. Language: Select English and press Enter.
  3. Installer update: Choose Continue without updating.
  4. Keyboard layout: Leave as English (US) and select Done.
  5. Installation type: Leave the default Ubuntu Server selected and choose Done. Do not select minimized.
  6. Network: VirtualBox assigns a 10.0.2.x address automatically via NAT. Select Done when the address appears.
  7. Proxy: Leave blank and select Done.
  8. Mirror: Wait for the installer to test the package mirror. When you see mirror location passed test, select Done.
  9. Storage 📷 Screenshot: Choose Use an entire disk. Use the down arrow to scroll to Done and press Enter. Review the filesystem summary (look for your 25 GB VirtualBox hard disk split into three partitions), then select Done. Confirm with Continue.
  10. Profile 📷 Screenshot: Fill in the following fields using Tab to move between them, then select Done:
    • Your name: Your real name or a course alias.
    • Server's hostname: A short lowercase name, e.g., ubuntu-csc106. No spaces or capital letters.
    • Username: A lowercase username you will remember — this is the account you will use all semester.
    • Password: Choose a password and type it twice. Write it down somewhere safe — you will type it frequently.
  11. Ubuntu Pro: Select Skip for now and press Enter to continue.
  12. OpenSSH: You may skip installing OpenSSH server for now — if you need it later in the course, it can always be installed with sudo apt install openssh-server. Select Done.
  13. Snaps: Do not select any optional featured server snaps. Use the down arrow to reach Done and press Enter.
  14. Wait for the installation to finish — approximately 5–15 minutes depending on your hardware. A spinning cursor at the bottom of the screen means it is still working. When you see Reboot Now at the bottom, select it and press Enter.

Part C — First Login and Verification

  1. After rebooting, if you see an error about failing to unmount the installation media, that is normal. Check Settings → Storage in VirtualBox — the optical drive should now show as empty. If it does not, click the optical drive entry and remove the ISO. Then press Enter (or use Input → Keyboard → Send Ctrl+Alt+Del) to continue the boot.
  2. Wait for the login prompt. It will show your server's hostname followed by login:.
  3. 📷 Screenshot: Type your username and press Enter, then type your password and press Enter. The password will not appear on screen as you type — this is normal for Linux.
  4. Once logged in you will see a brief system summary. This text-only interface is the normal Ubuntu Server environment — there is no graphical desktop. Servers are designed to be managed via the command line.
  5. Confirm network connectivity by pinging a public address:
    ping 8.8.8.8
    Press Ctrl+C to stop. You should see replies, confirming internet access through VirtualBox's NAT network.
  6. Update the package list to make sure everything is current:
    sudo apt update
    Enter your password when prompted. You may see a list of packages that can be upgraded — that is fine for now.
  7. To shut down the VM cleanly from inside the server:
    sudo poweroff
    Enter your password when prompted. The VM will shut down.
Tip — VM performance during install is slow? If the installation takes a very long time, try increasing RAM and CPU cores: power off the VM, go to Settings → System, and raise Memory to 8192 MB and Processors to 4. Then restart the install. As shown in the video, doubling resources can dramatically reduce install time.
