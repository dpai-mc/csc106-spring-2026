Setting Up Your Linux Environment

This tutorial prepares your Windows 11 computer for Lab 1. You will complete two tasks: install VirtualBox (a hypervisor that lets you run Linux inside a window on your PC), then use it to install Ubuntu Server 24.04 LTS as a virtual machine.

Each task includes a short video walkthrough and written step-by-step instructions. Use whichever works best for you — many students find it helpful to follow the written steps alongside the video.

On an older Intel-based Mac? Intel Macs use the same x86_64 processor architecture as Windows PCs. You can follow this guide and download VirtualBox for macOS (Intel) instead of the Windows version — all other steps are the same.
1

Install VirtualBox

What is a hypervisor?

A hypervisor is software that lets your computer run one or more separate operating systems inside isolated windows, as if each were running on its own hardware. The operating system running inside is called a virtual machine (VM). For this course, your Windows 11 desktop stays exactly as it is — VirtualBox simply adds a layer that runs Linux alongside it. Nothing about your existing system changes.

Oracle VirtualBox is free, open-source, and the standard hypervisor for Windows in this course. You will also install the VirtualBox Extension Pack, which adds support for USB devices, shared clipboard, and better keyboard/mouse integration inside your VM.

Download VirtualBox

Official download page: virtualbox.org/wiki/Downloads
✓ Download both items from this page: the Windows hosts installer and the VirtualBox Extension Pack (listed in a separate section just below the installer).
✓ Installer filename starts with VirtualBox- and ends in -Win.exe (~120 MB). The Extension Pack file ends in .vbox-extpack.

Written step-by-step instructions

  1. Open your browser and go to virtualbox.org/wiki/Downloads.
  2. Under VirtualBox platform packages, click Windows hosts to download the installer. Save it to your Downloads folder.
  3. On the same page, scroll down to the VirtualBox Extension Pack section and click All supported platforms to download it. Save it to the same Downloads folder.
  4. Open your Downloads folder and double-click the VirtualBox installer (filename starts with VirtualBox-).
  5. Click through the installer accepting all default settings. If Windows shows a User Account Control prompt, click Yes.
  6. A warning about network interfaces being briefly interrupted is normal — click Yes to proceed.
  7. Click Install and wait about 1–2 minutes for the installation to complete.
  8. When prompted to start VirtualBox at the end, leave the checkbox unchecked and click Finish.
  9. Return to your Downloads folder and double-click the Extension Pack file (ends in .vbox-extpack). VirtualBox opens automatically.
  10. In the Extension Pack installation dialog, scroll down to read the license agreement, then click I Agree. Wait about 30 seconds for the install to finish.
  11. To verify: in VirtualBox, click Tools in the left sidebar, then click Extensions. The Oracle VM VirtualBox Extension Pack should appear in the list and show as enabled.

Install Ubuntu Server 24.04 on VirtualBox

What is Ubuntu Server — and why no graphical desktop?

Ubuntu Server is a version of Linux built for computers that run services and host files rather than run desktop apps. It has no graphical interface — you interact entirely at the command line. This is exactly how professional Linux servers work in the real world. If the text-only login feels unfamiliar at first, that is completely normal and something you will quickly grow comfortable with over the course of the semester.

Why the ISO version matters: Because you are on a Windows PC (or an Intel Mac), your processor uses the x86_64 (amd64) architecture. You must download the amd64 ISO — the ARM version will not work on your hardware.

Download Ubuntu Server 24.04 LTS

Official download page: ubuntu.com/download/server
✓ Click Download Ubuntu Server 24.04 LTS. This is the correct x86_64 (amd64) version for Windows / VirtualBox.
✓ Filename looks like ubuntu-24.04.X-live-server-amd64.iso — approximately 2.7 GB. The download may take several minutes depending on your connection.

Written step-by-step instructions

Part A — Create the Virtual Machine

  1. Open VirtualBox. Click the New button (blue star icon in the toolbar) to open the new VM wizard.
  2. Enter a name for your VM, for example Ubuntu Server.
  3. Under ISO Image, click the dropdown and choose Other…. Browse to your Downloads folder and select the Ubuntu Server ISO file (ubuntu-24.04.X-live-server-amd64.iso).
  4. Important: Check the box labeled Skip Unattended Installation. This lets you walk through every installer screen yourself, which is required for the Lab 1 screenshots.
  5. Click Next. On the Hardware screen:
    • Base Memory: at least 2048 MB (2 GB); 4096 MB (4 GB) is recommended if your PC has the RAM.
    • Processors: 1 or 2 CPUs.
  6. Click Next. On the Hard Disk screen, set the virtual disk size to at least 25 GB. 50 GB is a comfortable size if you have the space.
  7. Click Next, review the summary, then click Finish. VirtualBox creates the VM and boots it automatically into the Ubuntu installer.

Part B — Walk Through the Ubuntu Installer

Use the arrow keys and Enter to navigate the installer. Steps marked 📷 Screenshot are required for the Lab 1 submission.

  1. Language: Select English and press Enter.
  2. Installer update: Choose Continue without updating.
  3. Keyboard layout: Leave as English (US) and select Done.
  4. Installation type: Leave the default Ubuntu Server selected and choose Done.
  5. Network: The installer configures your network via DHCP automatically. Select Done.
  6. Proxy: Leave blank and select Done.
  7. Mirror: Wait a moment for the installer to test the package mirror. When it shows OK, select Done.
  8. Storage 📷 Screenshot: Choose Use an entire disk. Select Done, then review the filesystem summary and select Done again. Confirm with Continue.
  9. Profile 📷 Screenshot: Fill in the following fields, then select Done:
    • Your name: Your real name or a course alias.
    • Server's hostname: A short lowercase name with no spaces, e.g., ubuntu-server.
    • Username: A lowercase username you will remember — this is the account you will use all semester.
    • Password: Choose a strong password and write it down somewhere safe. You will type it frequently.
  10. Ubuntu Pro: Select Skip for now.
  11. OpenSSH: Move the cursor to Install OpenSSH server and press Space to enable it. Select Done. OpenSSH allows you to connect to your VM from a terminal window later in the course.
  12. Snaps: Do not select any optional packages. Select Done.
  13. Wait for the installation to finish — approximately 5–10 minutes. When you see Reboot Now, select it and press Enter.

Part C — First Login and Verification

  1. If the system appears to hang after rebooting, go to the VirtualBox menu bar: Devices → Optical Drives → Remove disk from virtual drive, then press Enter in the VM window to continue booting.
  2. 📷 Screenshot: At the login prompt, type your username and press Enter, then type your password and press Enter. The password will not appear on screen as you type — this is normal for Linux.
  3. Once logged in, run the following command to confirm the system is working:
    uname -a
  4. 📷 Screenshot: The output should show the Linux kernel version and end with x86_64 GNU/Linux.
  5. Confirm network connectivity (you will need this for apt later):
    ip a
    You should see an IP address on one of the listed network interfaces.
  6. To shut down the VM cleanly when you are done:
    sudo shutdown now
    Enter your password when prompted.
Tip — VM reboots back into the installer? This means the ISO image was not ejected automatically. In VirtualBox, open Settings → Storage for the VM, click the optical drive entry showing the ISO, and remove it. Then start the VM again and it will boot normally into the installed system.
