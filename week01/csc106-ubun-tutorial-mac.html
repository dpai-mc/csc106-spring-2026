Setting Up Your Linux Environment

This tutorial prepares your Apple Silicon Mac (M1, M2, M3, or M4) for Lab 1. You will complete two tasks: install UTM (a hypervisor that lets you run Linux inside a window on your Mac), then use it to install Ubuntu Server 24.04 LTS as a virtual machine.

The tutorial is organized as follows: a single video walkthrough covers the entire process from start to finish, followed by two written task sections you can use alongside the video or on their own.

On an older Intel-based Mac? Intel Macs share the x86_64 architecture with Windows PCs. Follow the Windows 11 tutorial instead — install VirtualBox for macOS (Intel) and download the amd64 Ubuntu Server ISO. The steps are the same as the Windows guide.

Install UTM

What is a hypervisor?

A hypervisor is software that lets your computer run one or more separate operating systems inside isolated windows, as if each were running on its own hardware. The operating system running inside is called a virtual machine (VM). For this course, your macOS desktop stays exactly as it is — UTM simply adds a layer that runs Linux alongside it. Nothing about your existing system changes.

Apple Silicon Macs (M1 and later) use an ARM-based processor. UTM is a free, open-source hypervisor built specifically to take advantage of this architecture. It can virtualize ARM operating systems (like our Ubuntu Server ARM edition) at full native speed. Note that UTM is also available in the Mac App Store, but that version costs money — the free version from the UTM website is identical and the one we use in this course.

Download UTM

Official download page: mac.getutm.app
✓ Search for UTM and go to mac.getutm.app. Click Download to get the free .dmg file — not the Mac App Store link.
✓ The downloaded file is named UTM.dmg — approximately 20 MB.

Written step-by-step instructions

  1. Open your browser, search for UTM VM, and go to mac.getutm.app. Click Download to get the free .dmg file. Save it to your Downloads folder.
  2. Open Finder and go to your Downloads folder. Double-click UTM.dmg to mount the disk image.
  3. A window appears showing the UTM app icon and an Applications folder shortcut. Drag the UTM icon into the Applications folder to install it.
  4. If macOS asks whether to replace an existing copy, click Replace.
  5. Open Finder → Applications, scroll down to find UTM, and double-click it to launch. If macOS displays a security warning that the app was downloaded from the internet, click Open.
  6. UTM opens and displays your virtual machine library (empty on a fresh install). You are ready for Task 2.

Install Ubuntu Server 24.04 on UTM

What is Ubuntu Server — and why no graphical desktop?

Ubuntu Server is a version of Linux built for computers that run services and host files rather than run desktop apps. It has no graphical interface — you interact entirely at the command line. This is exactly how professional Linux servers work in the real world. If the text-only login feels unfamiliar at first, that is completely normal and something you will quickly grow comfortable with over the course of the semester.

Why the ISO version matters: Apple Silicon uses the ARM64 (aarch64) architecture. You must download the ARM64 Ubuntu Server ISO — the standard amd64 version will not run on your Mac. The ARM ISO is listed under "Alternative architectures" on the Ubuntu download page. Do not click the large green download button on the main page — that downloads the x86 version.

Download Ubuntu Server 24.04 LTS (ARM64)

Official download page: ubuntu.com/download/server/arm
✓ On the Ubuntu Server download page, click Alternative architectures, then click the ARM download for Ubuntu 24.04.x LTS. Do not click the large "Download Ubuntu Server 24.04 LTS" button — that downloads the amd64 (x86_64) version, which will not work on your Mac.
✓ Filename looks like ubuntu-24.04.X-live-server-arm64.iso — approximately 2 GB. The download may take around 5–10 minutes on a typical connection.

Written step-by-step instructions

Part A — Create the Virtual Machine in UTM

  1. Open UTM. Click the + button (Create a New Virtual Machine) to start the wizard.
  2. Select Virtualize — not Emulate. Because your Mac and Ubuntu Server both use ARM architecture, virtualization runs at full native speed.
  3. Select Linux as the operating system.
  4. Under Boot ISO Image, click the path field and browse to your Downloads folder. Select the ARM64 ISO you downloaded (ubuntu-24.04.X-live-server-arm64.iso). Click Continue.
  5. On the Hardware screen:
    • Memory: 4096 MB (4 GB) is the minimum recommended. 8192 MB (8 GB) will make the install noticeably faster if your Mac has the RAM.
    • CPU Cores: Leave at the default (UTM will choose a sensible value for your chip).
    • Leave Enable Hardware OpenGL Acceleration unchecked to avoid display issues with newer Linux drivers.
  6. On the Storage screen, leave the size at 64 GB (the default). Click Continue.
  7. Optional — Shared Directory: On the Shared Directory screen, click the path field and choose a folder to share between your Mac and the VM (e.g., your Downloads folder). Anything you put in that folder on your Mac will be accessible from inside the VM. Click Continue.
  8. Review the summary. Rename the VM if you like — for example, CSC106-Ubuntu. Click Save. The VM appears in the UTM library.
  9. Click the Play button (▶) to start the VM. The Ubuntu installer will boot inside the UTM window.

Part B — Walk Through the Ubuntu Installer

Use the arrow keys and Enter to navigate the installer. Steps marked 📷 Screenshot are required for the Lab 1 submission.

  1. After starting the VM, you may briefly see a "display output is not active" message — this is normal while the VM is booting. The installer will appear within a few seconds. Select Try or Install Ubuntu Server (the first option) and press Enter rather than waiting 30 seconds for it to auto-select.
  2. Language: Select English and press Enter.
  3. Installer update: Choose Continue without updating (use the current installer).
  4. Keyboard layout: Leave as English (US) and select Done.
  5. Installation type: Leave the default Ubuntu Server selected and choose Done.
  6. Network: UTM assigns a 192.168.64.x address automatically. This is UTM acting as a router (NAT) for your VMs. Select Done when the address appears.
  7. Proxy: Leave blank and select Done.
  8. Mirror: Wait for the installer to test the package mirror. When the message says the mirror location passed the test, select Done.
  9. Storage 📷 Screenshot: Choose Use an entire disk. Use the down arrow to scroll to Done and press Enter. Review the filesystem summary (your 64 GB virtual disk split into partitions), then select Done. Confirm with Continue.
  10. Profile 📷 Screenshot: Fill in the following fields using Tab to move between them, then select Done:
    • Your name: Your real name or a course alias.
    • Server's hostname: A short lowercase name with no spaces, e.g., ubu-server. No capital letters.
    • Username: A lowercase username you will remember — this is the account you will use all semester.
    • Password: Choose a password and type it twice. Write it down somewhere safe — you will type it frequently.
  11. Ubuntu Pro: Select Skip for now and press Enter to continue.
  12. OpenSSH: You may skip installing OpenSSH server for now — if you need it later in the course, it can always be installed with sudo apt install openssh-server. Select Done.
  13. Snaps: Do not select any optional featured server snaps. Use the down arrow to reach Done and press Enter.
  14. Wait for the installation to finish — approximately 5–7 minutes. When you see Reboot Now, select it with the arrow keys and press Enter.

Part C — First Login and Verification

  1. After clicking Reboot Now, UTM may not automatically eject the ISO. If the VM boots back into the installer, click the UTM stop button (square icon) to power it off.
  2. In UTM, click the VM name to select it, then click the Edit button (pencil icon). Go to the Drives section, find the CD/DVD entry that points to the ISO file, and click Clear (or delete that drive entry). Click Save.
  3. Click the Play button to start the VM again. It will now boot directly into the installed Ubuntu Server.
  4. You may briefly see the "display output is not active" message again during boot — this is normal. The login prompt will appear within a few seconds.
  5. 📷 Screenshot: Type your username and press Enter, then type your password and press Enter. The password will not appear on screen as you type — this is normal for Linux.
  6. Once logged in, you will see a brief system summary. This text-only interface is the standard Ubuntu Server environment — there is no graphical desktop. This is intentional: servers are managed via the command line and do not need a GUI.
  7. Confirm network connectivity:
    ping 8.8.8.8
    Press Ctrl+C to stop. You should see replies, confirming internet access through UTM's NAT network.
  8. To shut down the VM cleanly from inside the server:
    sudo shutdown now
    Enter your password when prompted. The VM will power off.
Tip — Install took too long? If the installation was very slow, try increasing the VM resources: power off the VM, click Edit in UTM, and raise Memory to 8192 MB and CPU cores to 4. As shown in the video, more RAM and CPU cores can make the install significantly faster.
