Setting Up Your Linux Environment

This tutorial prepares your Apple Silicon Mac (M1, M2, M3, or M4) for Lab 1. You will complete two tasks: install UTM (a hypervisor that lets you run Linux inside a window on your Mac), then use it to install Ubuntu Server 24.04 LTS as a virtual machine.

Each task includes a short video walkthrough and written step-by-step instructions. Use whichever works best for you — many students find it helpful to follow the written steps alongside the video.

On an older Intel-based Mac? Intel Macs share the x86_64 architecture with Windows PCs. Follow the Windows 11 tutorial instead — install VirtualBox for macOS (Intel) and download the amd64 Ubuntu Server ISO. The steps are the same as the Windows guide.
1

Install UTM

What is a hypervisor?

A hypervisor is software that lets your computer run one or more separate operating systems inside isolated windows, as if each were running on its own hardware. The operating system running inside is called a virtual machine (VM). For this course, your macOS desktop stays exactly as it is — UTM simply adds a layer that runs Linux alongside it. Nothing about your existing system changes.

Apple Silicon Macs (M1 and later) use an ARM-based processor. UTM is a free, open-source hypervisor built specifically to take advantage of this architecture. It can virtualize ARM operating systems (like our Ubuntu Server ARM edition) at full native speed, which is exactly what we need.

Download UTM

Official download page: mac.getutm.app
✓ Click Download on the UTM website to get the free .dmg installer. This is the recommended option for this course.
✓ UTM is also available in the Mac App Store, but that version costs money. The website version is identical and free.

Written step-by-step instructions

  1. Open your browser and go to mac.getutm.app. Click Download to get the free .dmg file. Save it to your Downloads folder.
  2. Open your Downloads folder and double-click UTM.dmg to mount it.
  3. A window opens showing the UTM icon and an Applications folder shortcut. Drag the UTM icon into the Applications folder.
  4. If macOS asks whether to replace an existing copy, click Replace.
  5. Eject the disk image by dragging it to the Trash or right-clicking and choosing Eject.
  6. Open Finder → Applications and double-click UTM to launch it. If macOS warns that the app was downloaded from the internet, click Open.
  7. UTM opens and shows your virtual machine library (empty on a fresh install). You are ready for Task 2.
2

Install Ubuntu Server 24.04 on UTM

What is Ubuntu Server — and why no graphical desktop?

Ubuntu Server is a version of Linux built for computers that run services and host files rather than run desktop apps. It has no graphical interface — you interact entirely at the command line. This is exactly how professional Linux servers work in the real world. If the text-only login feels unfamiliar at first, that is completely normal and something you will quickly grow comfortable with over the course of the semester.

Why the ISO version matters: Apple Silicon uses the ARM64 (aarch64) architecture. You must download the ARM64 Ubuntu Server ISO — the standard amd64 version will not run on your Mac. The ARM ISO is listed under "Alternative architectures" on the Ubuntu download page.

Download Ubuntu Server 24.04 LTS (ARM64)

Official download page: ubuntu.com/download/server/arm
✓ On the Ubuntu Server download page, look for Alternative architectures (or go directly to the ARM link above) and select the ARM64 download.
✓ Do not click the large "Download Ubuntu Server 24.04 LTS" button on the main page — that downloads the amd64 version, which will not work on your Mac.
✓ Filename looks like ubuntu-24.04.X-live-server-arm64.iso — approximately 2 GB. The download may take several minutes.

Written step-by-step instructions

Part A — Create the Virtual Machine in UTM

  1. Open UTM. Click the + button (or Create a New Virtual Machine) to start the new VM wizard.
  2. Select Virtualize — not Emulate. Because your Mac and Ubuntu Server both use ARM architecture, virtualization runs at full native speed. Emulation is only needed for non-ARM operating systems.
  3. Select Linux as the operating system.
  4. Under Boot ISO Image, click Browse and select the Ubuntu Server ARM64 ISO you downloaded (ubuntu-24.04.X-live-server-arm64.iso). Click Continue.
  5. On the Hardware screen:
    • Memory: 4096 MB (4 GB) is recommended. 8192 MB (8 GB) is fine if your Mac has the RAM.
    • CPU Cores: Leave at the default.
    • Leave Enable Hardware OpenGL Acceleration unchecked — it can cause display issues with some Linux drivers.
  6. On the Storage screen, the default is 64 GB — leave it as is. Click Continue.
  7. Optional: On the Shared Directory screen, you can click the path field and choose a folder (such as your Downloads folder) to share between your Mac and the VM. This is convenient for moving files in and out of Linux. Click Continue.
  8. Review the summary and click Save. The new VM appears in the UTM library.
  9. Click the Play button (▶) to start the VM. The Ubuntu installer boots inside the UTM window.

Part B — Walk Through the Ubuntu Installer

Use the arrow keys and Enter to navigate the installer. Steps marked 📷 Screenshot are required for the Lab 1 submission.

  1. A brief "display output is not active" message may appear while the VM boots — this is normal. The installer screen will appear within a few seconds.
  2. Language: Select English and press Enter.
  3. Installer update: Choose Continue without updating.
  4. Keyboard layout: Leave as English (US) and select Done.
  5. Installation type: Leave the default Ubuntu Server selected and choose Done.
  6. Network: The installer configures your network via DHCP automatically. Select Done.
  7. Proxy: Leave blank and select Done.
  8. Mirror: Wait a moment for the installer to test the package mirror. When it shows OK, select Done.
  9. Storage 📷 Screenshot: Choose Use an entire disk. Select Done, review the filesystem summary and select Done again. Confirm with Continue.
  10. Profile 📷 Screenshot: Fill in the following fields, then select Done:
    • Your name: Your real name or a course alias.
    • Server's hostname: A short lowercase name with no spaces, e.g., ubuntu-server.
    • Username: A lowercase username you will remember — this is the account you will use all semester.
    • Password: Choose a strong password and write it down somewhere safe. You will type it frequently.
  11. Ubuntu Pro: Select Skip for now.
  12. OpenSSH: Move the cursor to Install OpenSSH server and press Space to enable it. Select Done. OpenSSH allows you to connect to your VM from a Mac terminal window later in the course.
  13. Snaps: Do not select any optional packages. Select Done.
  14. Wait for the installation to finish — approximately 5–10 minutes. When you see Reboot Now, use the down arrow to select it and press Enter.

Part C — First Login and Verification

  1. After clicking Reboot Now, UTM may not automatically eject the ISO. If the VM boots back into the installer, power it off using the UTM toolbar stop button.
  2. In UTM, click the VM name to select it, then click Edit (the pencil icon). Go to the Drives section, find the CD/DVD entry pointing to the ISO file, and click the Clear button (or delete the drive entry). Click Save.
  3. Press the Play button to start the VM again. It will now boot into the installed system.
  4. A brief "display output is not active" message may appear again during boot — wait a few seconds for the login prompt.
  5. 📷 Screenshot: At the login prompt, type your username and press Enter, then type your password and press Enter. The password will not appear on screen as you type — this is normal for Linux.
  6. Once logged in, run the following command to confirm the system is working:
    uname -a
  7. 📷 Screenshot: The output should show the Linux kernel version and end with aarch64 GNU/Linux, confirming you are running the ARM64 build.
  8. Confirm network connectivity (you will need this for apt later):
    ip a
    You should see an IP address on one of the listed network interfaces.
  9. To shut down the VM cleanly when you are done:
    sudo shutdown now
    Enter your password when prompted.
Tip — Shared directory not visible inside Linux? If you set up a shared directory in UTM, you may need to mount it manually inside the VM. This is an optional convenience feature — you do not need it for Lab 1. The UTM documentation at docs.getutm.app covers this if you want to set it up later.
