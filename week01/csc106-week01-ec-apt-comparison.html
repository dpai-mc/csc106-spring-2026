Background: Two Package Management Families

Linux package managers fall into two dominant families, each with its own package format, low-level tool, and high-level front-end. Read Chapter 24 for the full treatment; this section is a quick orientation to help you get your bearings before you read.

Debian / Ubuntu Family

  • Package format: .deb
  • Low-level tool: dpkg
  • High-level front-end: apt
  • Repo config: /etc/apt/sources.list
  • Distributions: Debian, Ubuntu, Linux Mint, Kali

Red Hat / Fedora Family

  • Package format: .rpm
  • Low-level tool: rpm
  • High-level front-end: yum (older) / dnf (current)
  • Repo config: /etc/yum.repos.d/*.repo
  • Distributions: RHEL, CentOS, Fedora, Rocky, Alma
▶ Why does yum/dnf exist if rpm already works?

rpm can install a .rpm file directly, but it does not automatically resolve dependencies. If package A requires packages B and C, you would have to find and install B and C manually before A would work. This is tedious and error-prone at scale.

yum (and its modern replacement dnf) sits on top of rpm and adds automatic dependency resolution, repository management, and transaction history. The same logic applies on the Debian side: dpkg handles the low-level install/remove work, while apt manages dependencies and repositories.

▶ Quick command reference: apt vs. dnf side by side

Use this table as a reference while you read and write. Chapter 24 explains each command in detail.

Task apt (Debian/Ubuntu) dnf / yum (Red Hat/Fedora)
Refresh package index apt update dnf check-update
Upgrade all packages apt upgrade dnf upgrade
Install a package apt install <pkg> dnf install <pkg>
Remove a package apt remove <pkg> dnf remove <pkg>
Remove + config files apt purge <pkg> (config files removed by default)
Search for a package apt search <keyword> dnf search <keyword>
Show package details apt show <pkg> dnf info <pkg>
List installed packages dpkg -l rpm -qa
Files installed by a package dpkg -L <pkg> rpm -ql <pkg>
Which package owns a file dpkg -S <file> rpm -qf <file>

Note on yum vs. dnf: dnf is the current default on Fedora and RHEL 8+. yum is still found on older CentOS 7 systems and some documentation. The syntax is nearly identical — most yum commands work as-is with dnf.
