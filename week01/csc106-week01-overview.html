Welcome to CSC 106, Introduction to Linux II. This is where things get hands-on right away.

Key Concepts and Themes

Installation Decisions Matter

When you install Ubuntu Server, you are not just clicking "Next" repeatedly. You are choosing a disk layout, creating your first user account, and configuring network settings. These choices affect how the server behaves for its entire lifetime. Pay attention to why each option exists, not just what to click.

Package Management with apt

On Debian-based systems like Ubuntu, apt is how you install, update, and remove software. It pulls packages from repositories, resolves dependencies automatically, and keeps track of what is installed. You will use apt constantly throughout this course, so getting comfortable with it now pays off quickly. A common early mistake is forgetting to run apt update before trying to install something. That is normal; you will remember after the first time it trips you up.

Where Things Live on a Linux System

Linux organizes files in a very specific directory structure. Configuration files go in /etc, variable data (logs, package caches) goes in /var, and the /proc filesystem gives you a window into running processes and hardware. Understanding this layout helps you troubleshoot problems and find what you need without guessing.

Course Arc

CSC 106 is an 8-week course that builds from installation to a fully configured, secured Linux server. Here is the big picture:

  • Week 1 (this week): Install Ubuntu Server and learn package management.
  • Week 2: Explore hardware, the kernel, and the boot process.
  • Week 3: Understand runlevels, systemd targets, and managing services.
  • Week 4: Advanced shell scripting and Mini-Project 1 (automation script).
  • Week 5: Users, groups, and system security fundamentals.
  • Week 6: File permissions, ownership, and networking basics.
  • Week 7: Network configuration, web server setup, and security hardening.
  • Week 8: Integration and Mini-Project 2 (build and document a complete server).

Everything you do this week (your VM, your package management skills, your understanding of where files live) carries forward. The VM you create now is the same one you will configure, script, and secure over the coming weeks.

The course assesses three Student Learning Outcomes:

  • CSLO 1: Install and configure a Linux operating system.
  • CSLO 2: Configure a Linux operating system as a platform for web-based and other applications.
  • CSLO 3: Describe Linux runlevels and startup scripts.

This week focuses on CSLO 1. You will see CSLOs 2 and 3 introduced in the weeks ahead.

Looking Ahead: Week 2

Next week you will look under the hood of the system you just installed. You will learn what the Linux kernel does, how to inspect your hardware from the command line, and how your server gets from a powered-off state to a login prompt (the boot process). Commands like lspci, lsblk, lsmod, and dmesg will become part of your toolkit.

Having a working Ubuntu Server VM this week is essential. Week 2 assumes you can log in and run commands on it, so make sure your installation is solid before you move on.
