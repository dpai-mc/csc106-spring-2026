Week 7: Network Configuration, Server Setup, and Security

Connecting your system to the network and hardening it for service

Welcome to Week 7. This is the week where your Ubuntu Server VM stops being an isolated box and becomes something that could actually serve traffic on a network. You will configure networking with Netplan, set up a basic server service, schedule automated tasks with cron, and apply firewall rules to lock things down.

If the past few weeks felt like building the engine and interior of a car, this week you are putting it on the road and making sure the doors lock.

Learning Goals

By the end of this week, you should be able to:

  • Configure a static IP address on Ubuntu Server using Netplan
  • Explain how DNS resolution works on a Linux system and configure it
  • Set up a basic network service (such as a web server or SSH) and verify it is accessible
  • Use cron and crontab to schedule recurring tasks
  • Configure UFW (Uncomplicated Firewall) to allow or deny traffic by port and protocol
  • Apply basic security hardening practices to a Linux server

Key Concepts and Themes

As you work through this week, pay particular attention to these ideas:

  • Netplan and YAML configuration: Ubuntu Server uses Netplan for network configuration. The config files are in YAML format, which means indentation matters. A misplaced space can break your network. Take your time with these files.
  • Static vs. dynamic IP addressing: In Week 1, your VM used DHCP. This week you assign a static IP, which is what real servers need so clients can reliably find them.
  • DNS resolution: Understanding /etc/resolv.conf and systemd-resolved helps you troubleshoot "I can ping an IP but can't reach a hostname" problems.
  • Cron scheduling: The crontab syntax (minute, hour, day-of-month, month, day-of-week) looks cryptic at first, but it is one of those things that clicks once you write a few entries. Practice reading cron expressions out loud: "At minute 30, every hour, every day..."
  • Firewall as default-deny: UFW makes firewall management approachable, but the underlying concept is important: start by blocking everything, then open only what you need. This is a security mindset, not just a tool.
  • Defense in depth: A firewall is one layer. Keeping software updated, disabling unused services, and using strong authentication are other layers. No single measure is enough on its own.

Looking Ahead

Week 8 is the final week of the course, and it is entirely devoted to Mini-Project 2: building and documenting a complete Linux server from scratch. Everything you configure this week (networking, services, firewall, cron) feeds directly into that capstone project.

Think of this week as your dress rehearsal. The server configuration assignment gives you hands-on practice with exactly the skills you will need to demonstrate in Week 8. If you run into trouble with Netplan or UFW this week, that is the time to ask questions and work through it. By Week 8, you will want these skills to feel familiar so you can focus on building a polished, well-documented final deliverable.

Course Arc

The 8-Week Path

  • Week 1: Install Ubuntu Server, learn package management
  • Week 2: Explore the kernel, hardware, and boot process
  • Week 3: Manage services and startup with systemd
  • Week 4: Advanced shell scripting (Mini-Project 1)
  • Week 5: Users, groups, and system security
  • Week 6: File permissions and ownership
  • Week 7 (you are here): Networking, server setup, cron, and firewall
  • Week 8: Mini-Project 2 (capstone server build)

Weeks 1 through 3 built your system from the ground up. Week 4 taught you to automate tasks with scripts. Weeks 5 and 6 gave you control over who can access what. Now, Week 7 connects your server to the outside world and secures it. Week 8 pulls all of these threads together into one final project.

  • CSLO 2: Configure a Linux operating system as a platform for web-based and other applications. (Setting up a network service and making it accessible is exactly this.)
