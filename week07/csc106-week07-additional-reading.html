Week 7 Additional Reading

CSC 106 – Introduction to Linux II  ·  Week 7  ·  Server Configuration and Security

Week 7 is the most hands-on week of the course, and it is also the week with no assigned Cisco Networking Academy chapter. That means the resources below are not supplemental — they are the primary technical references for everything you will configure this week. You will be returning to them repeatedly as you work through the Server Configuration assignment.

The resources are organized by topic to match the five steps of Part A and the one step of Part B in Lab 5. You do not need to read every article in full before starting. Scan each one to understand what it covers, then consult the relevant resource as you work through each step. The DigitalOcean tutorials in particular are written for beginners and are excellent starting points if official documentation feels too dense.

How to use these resources
  • Before Part A, Step 2 (Static IP): Read the Ubuntu Netplan guide and check the Netplan.io reference for YAML syntax.
  • Before Part A, Step 3 (Nginx): Skim the Ubuntu Nginx guide to understand the installation and service layout. The Nginx Beginner's Guide explains the configuration file structure.
  • Before Part A, Step 4 (ufw): The DigitalOcean ufw tutorial is the recommended starting point — it is beginner-friendly and walks through every command in order.
  • Before Part A, Step 5 (Cron): Open Crontab Guru in a browser tab alongside your terminal so you can verify schedule expressions as you write them.
  • Before Part B (SSH Keys): The DigitalOcean SSH keys tutorial covers the exact workflow Part B requires.
  • For extra credit: The two TLS resources at the bottom of this page are the assigned reading for the Self-Signed TLS extra credit assignment.
Topic 1

Static IP Configuration with Netplan

Ubuntu Server Guide: Network Configuration (Netplan)

The official Ubuntu documentation for configuring network interfaces with Netplan, including examples for static IP addresses, DHCP, and DNS server configuration. This is the authoritative reference for the YAML file you will write in Part A, Step 2.

ubuntu.com/server/docs/network-introduction
Connects to: Lab 5, Part A — Step 2 (Configure Static IP with Netplan). Refer to the "Static IP addresses" section for the exact YAML structure and the netplan apply workflow.

Netplan Reference Documentation

The complete YAML schema reference from the Netplan project itself, documenting every supported configuration field including addresses, routes, nameservers, and dhcp4. Use this when you want to understand exactly what a field does or verify that your YAML syntax is correct.

netplan.io/reference
Connects to: Lab 5, Part A — Step 2. Useful when troubleshooting a failed netplan apply or when you want to understand what each field in the YAML file controls.
Topic 2

Installing and Configuring Nginx

Ubuntu Server Guide: How to Install Nginx

Ubuntu's official guide to installing and running Nginx as a web server, including service management with systemctl, the location of the web root and configuration files, and how to create a basic server block. This document covers the same steps you will follow in Part A, Step 3 of the lab.

ubuntu.com/server/docs/how-to-install-nginx
Connects to: Lab 5, Part A — Step 3 (Install and Configure Nginx). Reference this guide for the apt install command, service status checks, and understanding where Nginx stores its configuration and content files.

Nginx Beginner's Guide

A concise introduction to Nginx from the Nginx project itself, explaining how Nginx processes requests, where its main configuration file lives, and how to set up a basic HTTP server block. Helpful if you want to understand how Nginx works beneath the commands, or if you plan to customize it beyond the defaults.

nginx.org/en/docs/beginners_guide.html
Connects to: Lab 5, Part A — Step 3, and the Week 7 course theme of understanding what a web server actually does. Also lays groundwork for the extra credit TLS configuration.
Topic 3

Firewall Configuration with ufw

Ubuntu Server Guide: Firewalls

The official Ubuntu documentation for ufw, covering how to enable it, set default policies, allow and deny ports, use application profiles, and check firewall status. This is the reference that mirrors the exact commands in Part A, Step 4.

ubuntu.com/server/docs/firewalls
Connects to: Lab 5, Part A — Step 4 (Configure the Firewall with ufw). The application profiles section explains how to use ufw allow 'Nginx HTTP' as an alternative to specifying port numbers directly.

DigitalOcean: How To Set Up a Firewall with UFW on Ubuntu

A step-by-step tutorial that walks through ufw from scratch in a beginner-friendly way — setting default policies, opening ports, enabling the firewall, and verifying the configuration. If the official Ubuntu docs feel too terse, start here instead.

digitalocean.com — How To Set Up a Firewall with UFW on Ubuntu
Connects to: Lab 5, Part A — Step 4. This tutorial is especially useful for understanding the critical ordering requirement: allow SSH before enabling the firewall. It explains clearly what happens if you do it in the wrong order.
Topic 4

Scheduled Tasks with Cron

Crontab Guru — Interactive Cron Schedule Builder

An interactive web tool where you type a cron schedule expression and instantly see in plain English when it will run. Keep this open in a browser tab while writing your cron job — it is the fastest way to verify that your schedule expression does what you intend.

crontab.guru
Connects to: Lab 5, Part A — Step 5 (Set Up the Health-Monitoring Cron Job). Use it to verify 0 * * * * means "at the top of every hour" and to experiment with other schedule expressions.

Ubuntu Manpage: crontab(5) — Crontab File Format

The official manual page for the crontab file format, documenting the five time fields, special strings like @hourly and @reboot, and how environment variables are handled inside cron jobs. The authoritative reference when you need to understand exactly how cron interprets an entry.

manpages.ubuntu.com — crontab(5)
Connects to: Lab 5, Part A — Step 5. Particularly useful if your cron job runs but produces unexpected output, or if you want to use the @hourly shorthand instead of 0 * * * *.
Topic 5

SSH Key Authentication

DigitalOcean: How To Set Up SSH Keys on Ubuntu

A clear, step-by-step walkthrough of generating an SSH key pair with ssh-keygen, copying the public key to a server using ssh-copy-id, and verifying that passwordless login works. This is the primary reference for Part B of Lab 5 and covers both the conceptual "why" and the practical "how."

digitalocean.com — How To Set Up SSH Keys on Ubuntu
Connects to: Lab 5, Part B — all three steps. Read this before generating your key pair to understand where each file goes and why the permissions on ~/.ssh/ and authorized_keys must be set correctly.

Ubuntu Server Guide: OpenSSH Server

Ubuntu's official documentation for the OpenSSH server, covering installation, key-based authentication configuration, the sshd_config file, and common hardening steps like disabling password authentication. Useful as background reading for Part B and as a preview of the SSH hardening you will do in Week 8's Mini-Project 2.

ubuntu.com/server/docs/openssh-server
Connects to: Lab 5, Part B, and Week 8 Mini-Project 2. The sshd_config options covered here — particularly PasswordAuthentication no and PermitRootLogin no — are the hardening steps Week 8 will require.

Self-Signed TLS for Nginx (5 points)

These two resources are the assigned reading for the Week 7 extra credit assignment. To earn the 5 points, read both references, generate a self-signed TLS certificate using openssl, configure Nginx to serve HTTPS on port 443, update your ufw rules to allow port 443, and document the process with screenshots and explanation. Submit as an addendum to your Lab 5 document.

DigitalOcean: How To Create a Self-Signed SSL Certificate for Nginx on Ubuntu

A step-by-step tutorial covering how to generate a self-signed TLS certificate with openssl, configure Nginx to use it for HTTPS, and test that the secure connection is working. This is the primary hands-on guide for the extra credit assignment.

digitalocean.com — Self-Signed SSL Certificate for Nginx on Ubuntu

Nginx Official Documentation: Configuring HTTPS Servers

The Nginx project's own documentation on TLS/SSL configuration, covering the ssl_certificate and ssl_certificate_key directives, HTTPS server blocks, and session optimization. A more technical complement to the DigitalOcean tutorial that explains why each Nginx directive is needed.

nginx.org — Configuring HTTPS Servers
Note

What to Do If a Site Is Unreachable

External websites occasionally go offline or become temporarily unreachable. Before assuming a link is broken, use an online availability checker to determine whether the problem is on your end or the site's end. DownDetector is a widely used tool for this — it shows real-time outage reports from users worldwide for hundreds of services and websites.

downdetector.com — Website and Service Status Checker

If a resource on this page is temporarily unavailable, try again later. Most of the resources listed here are official documentation from Ubuntu, Nginx, and DigitalOcean — sites with high uptime. If a link appears to be permanently broken, let your instructor know through Canvas.
